# 산림경영 Wiki — 운영 스키마

## 세션 시작 시 필수 실행

**새 대화를 시작할 때마다 가장 먼저** 다음 명령을 실행하여 최신 wiki 내용을 가져온다:

```bash
git pull origin main
```

pull 후 변경된 파일이 있으면 사용자에게 한 줄로 알린다. 변경 없으면 별도 언급 불필요.

---

## 목적

산림청 지침·행정규칙을 기반으로 조림·숲가꾸기·벌채·산림계획 실무자가 Claude에게 질문하고 즉시 판단 근거를 얻을 수 있는 **팀 공유 위키**다.

## 레이어 구조

```
FM wiki/
├── raw/                   ← 원본 레이어: 불변 소스 데이터 (LLM이 읽기만)
│   ├── admrule-kr/        ← legalize-kr/admrule-kr sparse clone (산림청 행정규칙)
│   ├── legalize-kr/       ← legalize-kr/legalize-kr sparse clone (산림청 관련 법률)
│   ├── manuals/           ← 매뉴얼·편람·지침서 원본 마크다운 (git 추적, 협업자 공유)
│   └── README.md
├── wiki/                  ← 위키 레이어: LLM이 작성·관리
│   ├── index.md           ← 전체 페이지 카탈로그 (필수 유지)
│   ├── log.md             ← 인제스트·쿼리·린트 이력 (append-only)
│   ├── 행정규칙/          ← 원문 wiki화: 행정규칙 원문 → 마크다운 (법령별)
│   ├── 산림작업/          ← 합성 페이지: 실무 기준·절차 정리 (사업별)
│   ├── 산림계획/          ← 합성 페이지: 경영계획·수확조절 (사업별)
│   └── 매뉴얼/            ← 매뉴얼 wiki화: raw/manuals/ → 챕터별 마크다운
└── CLAUDE.md
```

**원칙**: `wiki/행정규칙/`은 원문 충실 변환, `wiki/산림작업/`·`wiki/산림계획/`은 실무 관점 합성. `wiki/매뉴얼/`은 산림청 발행 매뉴얼·편람을 챕터 단위로 분할하여 wiki화. 모두 교차참조하되 섞지 않는다. `raw/`의 소스는 불변 원본이므로 LLM이 읽기만 하고 수정하지 않는다.

### 4. 매뉴얼 인제스트 (매뉴얼·편람·지침서 추가)

1. 원본 마크다운을 `raw/manuals/{파일명}.md`로 저장
2. `wiki/매뉴얼/{매뉴얼명}/` 하위에 챕터별 페이지 생성
3. `wiki/index.md` 매뉴얼 섹션에 항목 추가
4. `wiki/log.md` 맨 아래에 항목 추가: `## [YYYY-MM-DD] ingest | {매뉴얼명}`

## 워크플로

### 1. 인제스트 (새 행정규칙 추가)

1. 원문 마크다운을 `wiki/행정규칙/{종류}/{규칙명}.md` 또는 챕터 분할 형식으로 저장
2. 관련 합성 페이지(`wiki/산림작업/` 또는 `wiki/산림계획/`) 내용 업데이트
   - 기존 내용과 충돌 시 → 새 기준으로 갱신하고 변경 사유 주석 추가
3. `wiki/index.md` 해당 항목 추가 또는 업데이트
4. `wiki/log.md` 맨 아래에 항목 추가: `## [YYYY-MM-DD] ingest | {규칙명}`

### 2. 쿼리 (사용자 질문 답변)

1. `wiki/index.md` 읽기 → 관련 페이지 파악
2. 관련 페이지 읽기 → 답변 합성
3. 답변에 **페이지명과 조항** 인용 (예: `wiki/산림작업/조림/지침.md §3.1`)
4. 복잡한 분석·비교 결과가 재사용 가치가 있으면 새 페이지로 저장 후 index 등록

### 3. 린트 (위키 점검)

다음 항목을 점검하고 결과를 `wiki/log.md`에 기록:
- 교차참조 깨진 링크
- 고아 페이지 (index에 없는 파일)
- 날짜 기준 갱신이 필요한 페이지 (행정규칙 개정 이후 미반영)
- 중요 개념이 본문에 등장하나 전용 페이지가 없는 경우

## 페이지 템플릿

### 합성 페이지 (산림작업·산림계획)

```markdown
# {제목}

> 근거: {행정규칙명} {조항}, {시행일}

## 핵심 기준

{표 또는 목록}

## 주요 절차

{단계별 설명}

## 자주 묻는 판단

| 상황 | 적용 기준 | 근거 |
|------|-----------|------|

## 관련 페이지

- [[{페이지명}]] — {관계 설명}
```

### 행정규칙 원문 페이지

```markdown
# {행정규칙명}

> 종류: {훈령/예규/고시/지침}  
> 발령: {발령번호}, {발령일}  
> 시행: {시행일}  
> 출처: {URL 또는 admrule-kr 경로}

{본문 — 원문 충실하게}

---

## 관련 wiki

- [[{합성 페이지}]] — {관련성}
```

## index.md 관리 규칙

- 모든 `.md` 파일(README, index, log 제외)은 index에 등록
- 형식: `- [제목](경로) — 한줄 요약`
- 카테고리: 산림작업 / 산림계획 / 행정규칙(고시·예규·지침·훈령)
- 인제스트·삭제 시마다 즉시 업데이트

## log.md 관리 규칙

- append-only — 기존 항목 수정 금지
- 항목 형식: `## [YYYY-MM-DD] {동작} | {제목}`
  - 동작: `ingest` / `query` / `lint` / `update`
- 항목 본문: 변경된 페이지 목록, 주요 판단, 발견한 모순

## 행정규칙 우선순위

같은 사항에 대해 여러 규칙이 충돌할 때:

1. 법률 > 시행령 > 시행규칙
2. 훈령 > 예규 > 고시 > 지침
3. 최신 발령일 우선
4. 충돌 발견 시 합성 페이지에 명시: `> ⚠️ {규칙A}와 {규칙B} 간 기준 상이 — {설명}`
