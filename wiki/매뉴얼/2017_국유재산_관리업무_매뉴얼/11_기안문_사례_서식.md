# Ⅺ. 기안문·사례·서식 활용법

> 원문: 2288~6840행의 실제 처리문서와 법정·내부 서식

## 원문 수록 문서의 가치

원문은 단순 설명보다 실제 업무에서 어떤 순서로 문서를 생산하고 어떤 첨부물을 요구했는지를 보여준다. 따라서 문장 자체보다 **문서 간 선후관계와 증거 묶음**을 학습하는 자료로 가치가 크다.

## 문서군

| 업무 | 대표 문서 |
|---|---|
| 사용허가·대부 | 접수·보완·반려, 의견조회, 현지조사, 검토보고, 허가서·계약서 |
| 산지 인허가 | 일시사용·전용 협의, 의제협의, 복구설계 승인 |
| 기간·권리 | 기간갱신, 권리양도, 명의변경, 반환 |
| 위반 | 실태조사, 사전통지, 청문, 취소·소멸 통지 |
| 처분 | 매각·교환 검토, 감정평가, 일상감사, 계약·등기 |
| 취득 | 사유림 매수 검토, 감정평가, 매매계약, 등기촉탁 |
| 무단점유 | 사실확인, 현황측량, 변상금 사전통지·부과, 포기·철거 각서 |

## 표준 문서로 재사용할 때

1. 수신·발신·기관명·연락처·개인정보를 제거한다.
2. 2017년 조문·기관명·서식번호를 현행 기준으로 교체한다.
3. 사례의 사실관계를 일반화하고 실제 사건명·지번을 사용하지 않는다.
4. 처분문에는 근거, 인정사실, 판단, 처분내용, 이행기한, 불복방법을 분리한다.
5. 첨부목록을 체크리스트로 바꾸어 누락을 방지한다.

## 권장 사건철 구조

1. 신청·민원 원본
2. 재산·권리·경계 자료
3. 법령검토 및 관계기관 협의
4. 현지조사·측량·사진
5. 가격·사용료·보증 산정
6. 내부결재·심의·감사
7. 처분서·계약서
8. 시스템 등록·수납·등기
9. 실태조사·변경·갱신
10. 반환·복구·종결

## 이미지 처리

원문 이미지가 문서 변환 당시의 임시 로컬 경로를 가리켜 현재 복원할 수 없다. 이 위키에서는 깨진 링크를 반복하지 않고 업무 흐름을 표·체크리스트로 재작성한다. 원본 HWP/HWPX/PDF가 확보되면 도면·화면 중 판단에 필요한 것만 선별 복원한다.
